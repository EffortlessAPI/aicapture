Command Line Arguments

There are 23 command line arguments defined at this time. They are:

-help or -h [bool]

Show help about how to use the ssotme cli

-init [bool]

Initialize the current folder as the root of an SSOT.me project

-install [bool]

Saves the current command into the SSoTmeProject file

-uninstall [bool]

Removes the current command from the SSoTme project file

-build or -b [bool]

Build any transpilers in the current folder (or children).

-buildAll or -ba [bool]

Builds all transpilers in the project

-describe or -d [bool]

Describes the current SSoTme project (and all transpilers)

-input or -i [string]

Input filename or comma separated list of file names

-output or -o [string]

Output filename

-account or -a [string]

The account which the transpiler belongs to

-parameters or -p [List of string]

A list of parameters

-listSettings or -ls [bool]

List of project settings

-addSetting or -as [List of string]

Adds a setting to the SSoTmeProject

-removeSetting or -rs [List of string]

REmoves a setting from the ssotme project

-keyFile or -f [string]

The keyfile to use. By default it looks for ~/SSOT.me/ssotme.key.

-emailAddress or -e [string]

The email address for the account authenticating

-secret or -k [string]

The secret associated with that email address

-waitTimeout or -w [int]

The amount of time to wait for the command to continue

-clean or -c [bool]

Don't output the final results - instead, clean

-cleanAll or -ca [bool]

Don't output the final results - instead, clean

-skipClean or -sc [bool]

Don't clean the output before cooking

-runAs or -ra [string]

Run as this user (look for this user's key file)

-preserveZFS or -rz [bool]

Determines if the input should be preserved.
